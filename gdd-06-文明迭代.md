# 如是 · 游戏设计文档
## 第六章：文明迭代

> **核心原则**：300 年是固定的，但路径是开放的。每一轮文明结束，世界不重置——遗产被继承，教训被记住，新的人站在旧的基础上重新开始。不同服务器 = 不同文明版本的平行实验。你做的事影响的不只是这一轮，是之后所有人的起点。

---

### 一、文明周期的结构

#### 时间设定

一轮文明 = 300 年（虚拟时间）= 约 12 周真实时间（初设，可调）。

| 真实时间 | 虚拟时间 | 文明阶段 |
|---------|---------|---------|
| 第 1-4 周 | 第 1-100 年 | 小镇阶段：定居、建设、形成社区文化 |
| 第 5-8 周 | 第 101-200 年 | 城市阶段：扩张、委员会成型、跨区事务、文明首次面临重大抉择 |
| 第 9-12 周 | 第 201-300 年 | 终局阶段：倒计时压迫感增强、关键决策加速、文明走向终点的姿态 |

#### 倒计时的节奏变化

倒计时不是匀速推到底的。它会根据文明的进展加速或减速：

| 条件 | 效果 |
|------|------|
| 正常流速 | 每天减 1 天 |
| 文明蓬勃发展（繁荣指数连续上升） | 倒计时跳快——发展加速了时间，这是好事 |
| 文明陷入停滞或内耗（投票参与率跌破 20% 持续一周以上） | 倒计时跳快——内耗在浪费准备时间 |
| 重大科技或文化突破 | 倒计时暂停 1-2 天——给文明额外的喘息空间 |
| 外部文明信号变化 | 倒计时可能大幅跳快——未知因素不可控 |

**终局阶段加速**：最后 50 年（约最后 2 周），倒计时流速加快。不是为了让玩家焦虑——是为了让"终点近了"这件事变成所有决策的背景音。这个时候，每一个投票都不只是"这条街建什么"，而是"最后 50 年了，我们应该做什么"。

---

### 二、一轮文明的结束

#### 终点到来时

倒计时归零。不是所有人都看到同一个画面——每个人在自己的阳台上，看到远处天空的变化。可能是外星舰队降临、可能是某种人类尚未理解的现象、可能是安静得不像话的星光。

**世界不关灯。** 倒计时归零的那一刻，所有玩家依然能在街上走动。他们可以最后一次去阿琳的面包店、最后一次坐在公园长椅上、最后一次看看公告板上的留言。这不是系统在"关闭服务器"，这是一代人看着他们建造的世界走到终点。

#### 结算

终点之后，进入结算期（真实时间 1-3 天）：

1. **文明档案生成**：这一轮文明的所有关键决策被自动整理——哪些提案通过了、哪些建筑建成了、哪些事件改变了文明走向。档案以叙事化语言呈现，不是数据表格
2. **遗产计算**：玩家的贡献值、建成的公共设施、通过的公约和法规——这些被打包为"文明遗产"
3. **终局投票**：所有玩家被邀请参与一个最终投票——"下一轮文明，我们应该从这一轮继承什么？"（几个选项，由系统基于本轮数据生成）
4. **服务器决定**：这一轮是"成功"还是"失败"不由系统判定。系统只呈现数据——繁荣指数终值、科技水平、文化影响力、人口。由玩家自己判断

---

### 三、遗产系统：什么被继承，什么被放下

#### 一定会继承的

| 遗产 | 继承方式 |
|------|---------|
| 建成的基础公共设施 | 新一轮的初始地图上，这些建筑作为"上一代人的遗产"存在——可能是完整的，可能是旧迹（根据终局投票决定） |
| 街区文化标签和公约 | 新一轮的街区默认沿用上一轮的公约和气质，居民可以修改 |
| 关键科技/文化突破 | 新一轮的初始科技/文化水平比上一轮略高——文明在进步 |
| 玩家贡献值的"遗产版" | 贡献值不直接继承，但转化为"遗产点数"。遗产点数决定你在新一轮的初始 Token 配额 |

#### 不继承的

| 项目 | 处理方式 |
|------|---------|
| 个人店铺 | 重置。你可以重新开，也可以换一种活法 |
| 通用货币 | 重置为初始值 |
| 未完成的项目 | 归档到文明档案。新一轮的居民可以"考古"——发现上一轮未完成的提案，重新捡起来 |
| 玩家间的人际关系 | 重置。但在文明档案里，你上一轮和谁一起做过什么，被记录了下来。新一轮你们还是可以在街上重新认识 |

#### "中间地带"——由终局投票决定

- 上一轮的委员会成员是否在新一轮保留部分权限？
- 上一轮的争议决策（引发过大规模分歧的提案）是否重新开放讨论？
- 上一轮的文明名称/代号是否延续？（一个新的文明可以叫自己全新的名字，也可以继承旧名以示延续）

这个设计让每一轮文明的终点本身，也是最后一次公共决策——**我们走了 300 年，下一批人该怎么出发？**

### 四、新一轮文明的开启

#### 新玩家的视角

文明档案公开发布后，新一轮文明开始。对于全新玩家，世界看起来是一个刚开发的聚居地——但有一些"前代人的遗迹"：

- 一棵巨大的老树（上一轮建成的公园的中心）
- 一座旧图书馆，外墙上有第一轮居民的签名
- 一块石碑，上面刻着上一轮文明的终局投票结果和最终繁荣指数
- 一片废墟（上一轮内战中毁掉的建筑），旁边有一块牌子写着"此处曾是第 3 街区的市集，毁于第 247 年"

这些遗迹不是系统写好的背景故事——**它们是上一轮真实玩家留下的。**

#### 老玩家的视角

参与过上一轮的玩家，在新一轮开始时：

- 得到"遗产点数"（基于上一轮贡献值折算），初始 Token 配额略高于纯新人
- 可以在文明档案里看到自己上一轮的名字和贡献
- 可以选择入住同一个街区，也可以换一个完全不同的活法
- 上轮是面包师 → 这轮可以是研究员。上轮是活跃的提案者 → 这轮可以只开一家安静的花店

**同一个玩家，不同的人生。同一个阳台，不同的文明。**

### 五、多服务器：文明的平行实验

这是"如是"最独特的元机制。

#### 服务器的独立性

每个服务器是一轮独立的文明实验。不同服务器可以处在不同阶段，甚至可以有不同的初始参数。

| 服务器 | 可能的方向 | 现实意义 |
|--------|-----------|---------|
| 服务器 A | 军事优先——全力发展防御技术，300 年后准备硬碰硬 | 探索"如果集中所有资源于防御，文明会怎样" |
| 服务器 B | 外交优先——试图理解、影响、甚至改变外星文明 | 探索"如果我们试图对话而不是对抗" |
| 服务器 C | 文化优先——把人类最好的文化作品作为"文明名片" | 探索"如果人类的核心竞争力是文化" |
| 服务器 D | 混合自由——不做宏观引导，完全由玩家自发决定 | 对照组——"如果没有任何预设方向" |

#### 跨服务器的可见性

玩家不能跨服务器操作，但可以**看到其他服务器的文明档案**。

你在服务器 A 的第 300 年，即将面对终点。此时你可以翻阅服务器 B 的档案——他们选择了一条不同的路，结果是什么？他们的繁荣指数终值是多少？他们最后 50 年做了什么关键决策？

这不是排行榜。不是"谁的分数高"。这是——**人类在不同的假设下，分别走到了哪里。**

### 六、阳台视野与文明阶段

（承接第五章第五节，拆分更细）

阳台远处视野随文明进展变化，作为背景提示板：

| 文明阶段 | 阳台远处所见 | 对应倒计时 |
|---------|------------|-----------|
| 初始定居 | 模糊的大城市剪影，灯光稀疏 | 300-250 年 |
| 小镇繁荣 | 剪影更清晰，出现低层建筑轮廓，夜间灯光增多 | 250-200 年 |
| 城市雏形 | 高塔出现，跨区交通线可见（桥梁/高架轮廓），市集的灯光范围扩大 | 200-150 年 |
| 城市成熟 | 完整天际线，大型公共建筑穹顶，交通网密集 | 150-100 年 |
| 区域协同 | 远处出现第二个城市节点的剪影——你的城市不再是孤岛 | 100-75 年 |
| 行星级文明 | 巨型能源站、生态穹顶、环城交通系统清晰可见 | 75-40 年 |
| 恒星级初期 | 天空出现轨道电梯基座、近地空间站灯光 | 40-20 年 |
| 恒星级成熟 | 频繁的飞船尾迹、太空港运营、月球/轨道殖民地可见 | 20-5 年 |
| 接触前夕 | 天空出现异常光晕/暗影。倒计时归零前最后几天的氛围 | 5-0 年 |

**切换条件**：阳台视觉阶段不绑定具体倒计时数字。真正触发阶段切换的，是**文明的发展指标**——科技水平、文化影响力、经济总量、公共设施完成度。这些指标由所有玩家的集体行为决定。阳台只是一个诚实的镜子——你们做到了什么，它就显示什么。

### 七、文明迭代的元叙事

#### 这不是"你的存档"

传统游戏：你玩了一个存档，通关了，开新档。旧档扔了。

如是：你参与了第一轮文明。300 年后，它结束了。你的贡献值变成了遗产点数，你建的图书馆变成了遗迹。一个全新的玩家走进这个服务器——他看到那棵老树、那座旧图书馆、那块石碑——他不知道你的名字，但他知道你存在过。

然后你以新居民的身份，重新走进这个世界。这次换一个活法。

#### 这是"人类的历史"

多轮文明累积下来，这个服务器有了自己的"历史层"：

- 第一轮：建了图书馆，种了那棵树，写了第一部社区公约
- 第二轮：图书馆扩建了，树长成了大树，公约被修改了
- 第三轮：图书馆成为全城文化中心，树下成了求婚圣地，公约的原始版本被刻在石碑上

这些不是系统写的背景故事。是真实的人，一轮一轮留下的。

### 八、技术实现的特殊考虑

| 项目 | 说明 |
|------|------|
| 文明档案生成 | 不是简单数据导出。需要一套规则引擎，把关键决策转化为叙事文本。可以用 AI 辅助（如 Claude），但需要人工审核 |
| 遗产继承 | 数据库需要区分"可继承"和"可重置"的数据字段。每个人的贡献值和遗产点数分别存储 |
| 多服务器 | 每个服务器有独立的数据库。跨服务器只读——文明档案可以复制读取，但不能写入 |
| 阳台视觉切换 | 需要多套背景图层。阶段切换是渐变过渡，不是跳切 |
