# 如是 · 游戏设计文档
## 第七章：界面设计稿

> **核心原则**：界面不做推销员。不弹窗、不红点、不任务箭头、不下拉对话框。一切信息在那里，你选择去看。一切操作通过触碰和移动完成。

---

### 一、设计基调

#### 视觉语言

- **色调**：暖灰基底 + 自然材质色（木色、纸白、铜绿、墨色）。不冷，不平。像一本旧书的封面和一本新日程本的纸页之间的感觉
- **字体**：中文正文用衬线体（如思源宋体），数字和标签用等宽无衬线。中文的节奏感来自汉字本身的结构，不需要花哨排版
- **质感**：不是扁平，不是拟物。是"物质的减法"——日程本的纸纹、公告板的木质、App 界面的磨砂玻璃感。不炫耀质感，但触感是对的
- **动效**：呼吸级。翻转（日程本翻页）、渐显（文字出现）、轻微上浮（消息提示）。没有弹跳、没有晃动、没有"恭喜你"的花瓣

#### 美术风格：国画山水为骨

**整体画风**：宣纸为底，国画山水为骨。建筑是淡墨线条勾勒，街道是纸上洇开的墨迹。天空之城的云用留白表现——空白本身就是云。皮影人不是"放进"画里，是**生在纸上**——宣纸上的纸人，材质天然一体。

**色彩**：不是传统水墨的纯黑白。色彩可以更丰富——暖黄的灯光在淡墨街景中像一颗琥珀，老树的叶子可以是墨绿或赭石，傍晚的天空可以是淡胭脂，面包店的灯光是橘色的暖。有颜色，但不艳。是"墨色中点缀的色彩"，而不是"彩色上面画了几笔墨"。

**和精神内核的统一**：国画山水的美学——留白、写意、不填满、余韵——与如是底层设计哲学完全一致。日程本不命令你 = 画面不留满。皮影人背面写字 = 题跋。阳台 = 画框。倒计时在角落安静地走 = 画上的一枚小印。

### 二、阳台界面（场景一）

#### 布局

```
┌────────────────────────────────────────────┐
│                                            │
│         [远处城市天际线 + 倒计时数字]         │
│                                            │
│                                            │
│                                            │
│   ┌──────────────────────┐                │
│   │    阳台栏杆            │                │
│   │  [嵌入屏幕: 倒计时]    │                │
│   └──────────────────────┘                │
│                                            │
│   ┌──────────────┐                        │
│   │  日程本(摊开)  │    ☕ 一杯茶            │
│   │              │                        │
│   └──────────────┘                        │
│                                            │
│         [下楼 → 进入街区]                    │
└────────────────────────────────────────────┘
```

#### 交互元素

| 元素 | 交互 | 反馈 |
|------|------|------|
| 倒计时屏幕 | 不可交互，但跳秒时会变红/闪烁 | 视觉存在——你知道它在 |
| 日程本 | 点击翻开/合上 | 翻页动画，纸张质感 |
| 远处城市 | 不可交互 | 随着文明阶段变化（见第五章），视觉背景更新 |
| 下楼 | 点击画面底部或边缘 | 画面过渡到斜 45° 街区 |
| 转身进屋 | 点击阳台后方 | 进入住所内部（待设计） |

#### 氛围

- 没有 UI 叠层。画面是完整的阳台场景。
- 退出时没有"是否保存"——一切自动保存。你只是转身走进屋里。

### 三、日程本界面

#### 布局

打开日程本后的画面。全屏覆盖，背景是摊开的笔记本。

```
┌────────────────────────────────────────────┐
│                                            │
│   今日                      倒计时 109,574d  │
│   ───                                      │
│                                            │
│   天气：晴  16°C - 24°C                     │
│                                            │
│   ▸ 悬赏"西侧空地规划"投票今日开始           │
│     （截止后天 20:00）                       │
│                                            │
│   ▸ 阿琳的新品：黑麦酸面包（今日上架）        │
│                                            │
│   ▸ 林教授在公告板留言，想找人一起            │
│     整理街区旧书                             │
│                                            │
│   ─────────────────────────────             │
│   你昨日的踪影                               │
│   · 在 B-4 清理了杂草                       │
│   · 在阿琳的面包店买了全麦吐司               │
│   · 坐在公园长椅上待了一会儿                 │
│                                            │
│   ─────────────────────────────             │
│   街区此刻                                   │
│   · 面包店已营业                             │
│   · 公告板有 2 条新消息                      │
│                                            │
│                         [翻下一页 →]        │
└────────────────────────────────────────────┘
```

#### 翻页内容

- **今日** → 如上述
- **本周** → 进行中的悬赏、即将到来的事件、本周活跃度小结
- **倒计时追踪** → 倒计时剩余天数、本周减少天数、倒计时跳秒记录
- **文明档案**（城市阶段后解锁） → 本轮文明的关键决策时间线
- **你在"如是"的时间** → 你已在此多少天、留下过什么（时钟的另一面）

#### 交互

- 左右滑动翻页，或点击页面边缘
- 日程本上的每一项都是可点击的——点击"悬赏投票"→ 跳转到 App 的提案投票页。点击"阿琳的新品"→ 在地图上高亮面包店
- "你昨日的踪影"不可点击——它就是记录。只是告诉你昨天你去过哪

#### 设计约束

- 不出现"今日任务"标题
- 不出现红点、未读数字
- 文字是陈述句，不是命令句。不写"快去投票吧"，只写"投票今日开始"
- 倒计时在角落，小字。不抢夺注意力

### 四、斜 45° 街区界面（场景二）

#### 布局

全屏等距视角地图。底部无固定工具栏。所有 UI 通过场景内元素触发。

```
┌────────────────────────────────────────────┐

│    └──┘     └──┘              └────┘      │
│                                            │
│  ┌──┐  ┌──┐  ┌──┐     🌳       ┌──┐      │
│  │B4│  │B3│  │B2│    老树      │公│      │
│  └──┘  └──┘  └──┘   长椅       │告│      │
│  你的   杂货   空              │板│      │
│  地块   铺    地              └──┘      │
│                                            │
│       ┌──────┐                            │
│       │ 面包  │                            │
│       │  店   │                            │
│       └──────┘                            │
│                                            │
│               [← →  滑动移动]               │
│                                            │
│   [日程本]           [App]     [回住所]     │
└────────────────────────────────────────────┘
```

#### 交互

| 操作 | 效果 |
|------|------|
| 左右滑动 / 拖拽 | 在街区地图上横向移动视野 |
| 点击建筑 | 触发对应的功能——店铺打开商品列表、公告板打开信息流、长椅坐下 |
| 点击 NPC / 玩家 | 显示简短状态（"阿琳 · 面包店 · 正在营业"），如果距离近可以点头/对话 |
| 点击自己的地块 | 进入经营意向登记或店铺管理 |
| 底部三个按钮 | 日程本（翻开日程本）、App（打开街区通）、回住所（回到阳台） |

#### 地图渲染

- 当前可见范围 = 玩家的"视野距离"。新手只能看到自己所在的一段街，随着参与度提升视野逐渐放宽
- NPC 在地图上活动——阿琳早上在面包店，午饭后可能在公园长椅
- 天气和昼夜通过光照层和粒子（雨、雪、落叶）表现
- 倒计时在画面角落一个非常小的位置显示。没有大数字覆盖画面

#### 四个 Tab

**动态**（默认 Tab）：
- 进行中的悬赏卡片
- 公告板最新消息
- "街区上空"——玩家的非正式发言、想法、闲聊。不是朋友圈，是一句话的实时浮现。发一条 5 Token。24 小时后自动消失。这是低门槛的社区声音，不是内容沉淀

**提案**：
- 进行中的悬赏列表 + 各自的投票状态
- 玩家自提方案列表
- 历史提案档案（按时间线排列）
- "提交方案"按钮 → 进入方案撰写界面

**邻里**：
- 你在街上互动过的人的卡片（不是好友列表，是"你认识的人"）
- 每个卡片显示：名字、职业、最近一句话、你上次和 TA 互动是什么时候
- 不显示在线状态

**我的**：
- 我的经营意向和店铺状态
- 贡献值 + 投票权
- Token 余额（今日剩余 + 本周剩余）
- 通用货币余额
- 我的提案记录
- 我的投票记录

### 八、议事厅界面

**App 讨论区模式**（日常）：
- 每个悬赏/提案下面有讨论区
- 发言消耗 5 Token。发言记录旁显示消耗数
- 高质量的发言可以被"认可"——如果发言被认可，系统返还 10 Token（净赚 5）
- 讨论区不限字数。质量的信号是认可数，不是长度

**物理议事厅**（在街上点击议事厅建筑）：
- 进入建筑内部场景（室内 2D 画面）
- 墙上有数字板显示倒计时和历史提案数
- 桌子上摊着当前热门的讨论
- 旁听不消耗 Token
- 如果讨论区正在激烈辩论，议事厅上方在街景中升起一面小旗

### 十一、设计自查

- [x] 没有任何弹窗
- [x] 没有任何红点/未读标记
- [x] 没有任何任务箭头
- [x] 信息通过场景元素呈现（日程本、公告板、App 动态）
- [x] 所有交互通过触碰/移动，不使用对话选择
- [x] 日程本只陈述，不命令
- [x] App 的"街区上空"是轻量表达入口——低门槛但不泛滥（消耗 Token + 24 小时消失）
- [x] 提案是一张空白卡片，不是表单
- [x] 公告板保留下线内容——街区有记忆
- [x] 整体视觉：留白、克制、物质的减法
