## 预备：新建文件

1. 打开 Excel（或 WPS 表格）
2. 新建空白文件
3. 底部 Sheet1 改名为 `全局参数`（双击 Sheet1 标签 → 输入新名字 → 回车）
4. 点旁边的 `+` 新建 Sheet，改名为 `居民档案`
5. 同样方法创建：`每日流水`、`街区汇总`、`系统健康度`
6. 保存：`文件 → 另存为 → 如是-经济模拟.xlsx`

## Sheet 3：每日流水

这是最大的表。40 行居民 × 28 天。

### 第一天（Day 1）的列布局

Day 1 的列放在 A1-M1（13 列），Day 2 放在 N1-Z1（13 列），依此类推。28 天 × 13 列 = 364 列。

听起来多，但大部分列是拖公式自动生成的。你只需要做 **Day 1 的 13 列**，剩下的向右复制即可。

### Day 1 表头（第 1 行）

| 列 | 单元格 | 标题 |
|----|--------|------|
| A | A1 | ID |
| B | B1 | 街区 |
| C | C1 | 类型 |
| D | D1 | 职业 |
| E | E1 | 活跃概率 |
| F | F1 | D1-上线? |
| G | G1 | D1-Token余额(始) |
| H | H1 | D1-经营收入 |
| I | I1 | D1-基础分配 |
| J | J1 | D1-交易支出 |
| K | K1 | D1-Token消耗 |
| L | L1 | D1-Token余额(末) |
| M | M1 | D1-货币余额 |
| N | N1 | D1-贡献值变化 |
| O | O1 | D1-贡献值累计 |

> 实际是 15 列（A→O），不是 13。刚才记错了。Day 2 从 P 列开始。

### Day 1 公式（第 2 行开始，每行一个居民）

**前提**：先用 A2-A41 把 ID 拷过来，B2-B41 把街区拷过来，C2-C41 把类型拷过来，D2-D41 把职业拷过来，E2-E41 把活跃概率拷过来。

方法：在 A2 输入 `=居民档案!A2`，然后往右拖到 E2，再往下拖到 A41-E41。这样 Sheet3 的 A-E 列自动引用 Sheet2 的 A-E 列。

**上线?（F2）**：
```
=IF(RAND()*100 < E2, "是", "否")
```
> 含义：随机数 0-100，如果小于活跃概率，则为"是"，表示今天上线。
> 往下拖到 F41。

**Token 余额(始)（G2）**：
Day 1 初始 Token = 日配额 100 + 0（因为是第一天）
```
=TokenDay
```
> 往下拖到 G41。

**经营收入（H2）**：
```
=IF(F2="是", VLOOKUP(D2, Sheet2!J:K, 2, FALSE) * (0.7 + RAND()*0.6), 0)
```
> 含义：如果上线，查职业产出表，乘以 0.7~1.3 的随机系数（即 ±30%）。
> 往下拖到 H41。

**基础分配（I2）**：
```
=DailyGold
```
> 无论是否上线都发。
> 往下拖到 I41。

**交易支出（J2）**：
```
=IF(F2="是", RANDBETWEEN(5, 30), 0)
```
> 含义：上线则随机买 5~30 货币的东西。不上线不买。
> 往下拖到 J41。

**Token 消耗（K2）**：
```
=IF(F2="是", RANDBETWEEN(20, 80), 0)
```
> 含义：上线则今天因为投票/发言/提案总共消耗 20~80 Token。具体投了什么后面再细化，现在是简化版，保证跑得通。
> 往下拖到 K41。

**Token 余额(末)（L2）**：
```
=G2 - K2
```
> 如果结果为负 → 说明 Token 不够。后面优化时会加逻辑（不够 = 今天不能参与那么多活动）。
> 往下拖到 L41。

**货币余额（M2）**：
```
=StartGold + H2 + I2 - J2
```
> 往下拖到 M41。

**贡献值变化（N2）**：
```
=IF(AND(F2="是", E2=80, RAND()<0.1), BountyReward, 0)
```
> 含义：如果上线 + 积极型 + 10% 概率 → 获得悬赏采纳奖励 200 贡献值。这个概率很低是为了模拟真实情况（一周才 1 次悬赏，不是天天有人拿到奖励）。
> 往下拖到 N41。

**贡献值累计（O2）**：
```
=N2
```
> Day 1 的累计 = Day 1 的变化（因为初始为 0）。
> 往下拖到 O41。

### Day 2：复制 Day 1 并调整公式

Day 2 从 P 列开始。

**表头（第 1 行）**：把 Day 1 的表头复制过来，把 "D1" 改成 "D2"。

| P1 | Q1 | R1 | S1 | T1 | U1 | V1 | W1 | X1 | Y1 | Z1 | AA1 | AB1 | AC1 | AD1 |
|----|----|----|----|----|----|----|----|----|----|----|----|----|----|
| ID | 街区 | 类型 | 职业 | 活跃概率 | D2-上线? | D2-Token余额(始) | D2-经营收入 | D2-基础分配 | D2-交易支出 | D2-Token消耗 | D2-Token余额(末) | D2-货币余额 | D2-贡献值变化 | D2-贡献值累计 |

**关键区别公式**：
- **上线?（U2）**：同 Day1 `=IF(RAND()*100 < E2, "是", "否")` → 往下拖
- **Token 余额(始)（V2）**：`=L2 + TokenDay`  
  → 含义：昨天末的余额 + 今天赠送 100。如果今天是新的周一（如 Day 8），还要手动加 300
- **经营收入 ~ Token 消耗**：同 Day 1 公式，只是把 F2 改成 U2
- **货币余额（AB2）**：`=M2 + W2 + X2 - Y2` → 昨天货币 + 今天收支
- **贡献值累计（AD2）**：`=O2 + AC2` → 昨天累计 + 今天变化

然后 P2-AD2 往下拖到 P41-AD41。

---

### 后续 26 天

以此类推，每 15 列一组。每组的关键不同：
1. Token 余额(始) = 前一天余额(末) + TokenDay
2. 货币余额 = 前一天货币余额 + 今天变化
3. 贡献值累计 = 前一天累计 + 今天变化
4. 每 7 天（Day 7 结束后）：下一组第一天 Token 余额(始) 额外加 TokenWeek（300）

> **简化方案**：你先把 Day 1 到 Day 7 做出来，后面的告诉我"继续"我再引导你复制。一口气 28 天容易出错。

## Sheet 4：街区汇总

### 表头

| 单元格 | 内容 |
|--------|------|
| A1 | 街区汇总 |
| A3 | 街区 |
| B3 | 总人口 |
| C3 | D1-在线人数 |
| D3 | D1-总经营收入 |
| E3 | D1-总交易 |
| F3 | D1-总Token消耗 |
| G3 | D1-总贡献值 |
| H3 | D1-投票参与率 |

> C3-H3 是 Day 1 的汇总。Day 2 从 I3 开始，同理 28 天 × 7 列。

### 第 7 街区汇总（第 4 行）

- A4：`第7街区`
- B4：`=COUNTIF(每日流水!B2:B41, "第7街区")`
- C4：`=COUNTIFS(每日流水!B2:B41,"第7街区",每日流水!F2:F41,"是")`
  > 含义：统计第 7 街区 + 上线?="是" 的人数
- D4：`=SUMIFS(每日流水!H2:H41, 每日流水!B2:B41, "第7街区")`
  > 含义：加总第 7 街区所有人的经营收入
- E4：`=SUMIFS(每日流水!J2:J41, 每日流水!B2:B41, "第7街区")`
- F4：`=SUMIFS(每日流水!K2:K41, 每日流水!B2:B41, "第7街区")`
- G4：`=SUMIFS(每日流水!N2:N41, 每日流水!B2:B41, "第7街区")`
- H4：`=C4/B4`
  > 含义：在线人数 / 总人口 = 参与率

第 5 行（第 3 街区）、第 6 行（第 5 街区）：同理，把"第7街区"换成"第3街区"/"第5街区"。

### 验证 A：Token 充足度

| A5 | Token 充足度 |
| A6 | 日均 Token 消耗 < 日配额? |
| B6 | 平均消耗 < 70 Token |

如果要自动算：需要在每日流水上增加一列，算出每个玩家 28 天的平均 Token 消耗。

**简化做法（手动算）**：跑完 28 天之后，回到每日流水 Sheet，随便看几个 Token 消耗列，目测判断大部分人的消耗是不是在 20-80 之间。

### 验证 B：贡献值集中度

| A8 | 贡献值集中度 |
| A9 | TOP3 占总贡献值比例 |
| B9 | < 50% |

**公式**：需要在第 28 天的贡献值累计列取前 3 名，除以总贡献值。等跑完之后再做。

**简化**：看最后一天的贡献值累计列，目测判断是不是有极端值——一个人几千，其他人全是 0。

### 验证 C：经济通胀

| A11 | 经济通胀 |
| A12 | 28天后货币总量/初始总量 |
| B12 | < 3 倍 |

初始总量 = 40 人 × 500 = 20,000

28 天后总量 = Sheet3 第 28 天的货币余额列 全部加起来

### 验证 D：死号比例

| A14 | 死号比例 |
| A15 | 货币归零人数 |
| B15 | < 4 人（10%）|

28 天后，统计货币余额 ≤ 0 的人数。
