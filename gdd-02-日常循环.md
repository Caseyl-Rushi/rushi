# 如是 · 游戏设计文档
## 第二章：日常循环

> **核心原则**：时间有两面——正向流逝，反向提醒你找回些什么。用事件推动时间，而不是用时间规划事件。这个世界里，人工作不是为生存，是在实现自己。一切发生了，就去解决。

---

### 时间的基础设定

**一周 7 天**，与现实同步。不刻意划分工作日与周末。但遵循人的自然节奏——早晨热闹，午后慵懒，夜晚安静。某些日子约定俗成会比较活跃，但不是系统强制，是习惯。

**一天的时间流速**：虚拟世界的一天 = 真实世界的 24 分钟。4 分钟 = 1 小时。

（此数值为初设。阶段 1 纸面验证时调整。）

现实时间的节奏是这样：

| 现实时长 | 虚拟时间 | 光线 | 街区状态 |
|---------|---------|------|---------|
| 0-4 分钟 | 0:00 - 6:00 | 暗 → 渐亮 | 街区沉睡。少数窗口亮灯。无人走动 |
| 4-8 分钟 | 6:00 - 12:00 | 清晨 → 上午 | 烟火气。店铺开门。公告板可能更新。街上开始有人 |
| 8-12 分钟 | 12:00 - 18:00 | 午后 | 最热闹。市集活跃。长椅有人。事件多发 |
| 12-16 分钟 | 18:00 - 24:00 | 傍晚 → 深夜 | 街灯亮。议事厅可能开会。远处城市灯火通明。人们归家 |

玩家登录时，世界的时间停在**他上次离开时的虚拟时间**。每个人的世界是准同步的——你离线了，你街区里的事还在发生。但你的日程本会帮你补上。

---

### 一天的时间线（微观循环）

#### 清晨（虚拟 6:00 - 9:00）

玩家登录，站在阳台。清晨光线斜斜地照过来，街道上有薄雾。

日程本翻开。今天的内容已经更新——不是系统"推送"的，是昨天和昨天之前的世界动态，自然流到了今天：

```
今日
───

天气：晴
温度：16°C - 24°C

日程
· 悬赏"西侧空地规划"投票今日开始（截止后天 20:00）
· 阿琳的新品：黑麦酸面包（今日上架）
· 林教授在公告板留言，想找人和他一起整理街区图书馆的旧书

你昨日的踪影
· 在 B-4 地块清理了杂草
· 在阿琳的面包店买了全麦吐司
· 坐在公园长椅上待了一会儿
· 翻看了悬赏方案，尚未提交

街区此刻
· 面包店已营业
· 杂货铺正在开门
· 公告板有 2 条新消息
· 公园长椅上暂时没人
```

注意：没有"今日任务"。没有"建议你去做"。日程本只是告诉玩家——"这是你的世界，这是今天的样子。"

倒计时在这个页面的角落，安静地显示：**109,574 天 7 小时**。

---

#### 白天（虚拟 9:00 - 18:00）

街区进入最活跃的时段。玩家可以：

**在自己的经营用地上**：
- 开门营业。如果登记了经营意向、布置了店面，你的店铺在白天自动运转——产出通用货币，有客人进出
- 你的营业时间由你决定。你可以设定"只在下午开"，也可以"24 小时亮灯"。不设限
- 店铺状态影响街区繁荣指数中的经济指标

**在街上**：
- 去阿琳的面包店。买面包。和她聊两句——她可能提起今天的天气、昨天的客人、或者她对那个悬赏的想法
- 去杂货铺。补充日常用品，或者看到新到的货
- 去公告板。新的悬赏、新的提案、新的留言。有人在找猫，有人在招募一起整理图书馆的伙伴
- 去公园。长椅上可能有人——林教授在看书，陈医生在休息，或者另一个玩家在发呆

**在 App 上（可随时打开）**：
- 查看当前悬赏的进度
- 阅读提案的详细内容
- 看到街区里的最新动态——谁提交了新方案、谁的评论被很多人认可
- 查看自己的贡献值和投票权

**公共事件**（随机触发，非固定时间）：
- 一个旅行商人来到小镇，带来了 5 种外面的商品。街区只能买 2 种。需要投票决定
- 隔壁街区发来合作邀请
- 雨季提前到来，一些设施需要加固
- 委员会发布了一个紧急悬赏

这些事件不固定在周几的几点。它们**发生的时候就是发生的时候**。玩家打开日程本能看到"正在进行的公共事件"，如果已经错过了前期讨论，仍然可以参与投票或查看结果。

事件推动时间，不是时间规划事件。

---

#### 傍晚到夜里（虚拟 18:00 - 24:00）

街灯亮起。店铺开始关门。但有些地方还亮着——面包店可能还有最后的客人，议事厅里可能在讨论一个热点提案，公园长椅上有人坐着看星星。

玩家回到阳台。远处的城市灯火通明。日程本自动更新了今天的记录：

```
今日踪迹
───
08:15  阿琳的面包店：买了黑麦酸面包、和她聊了悬赏的事
10:32  公告板：阅读了"西侧空地"提案
12:47  公园长椅：坐了一会儿，林教授在旁边。他提了一句图书馆旧书的事
14:20  B-4 地块：花店营业了 3 小时，有 4 位客人
17:55  议事厅：参与了悬赏投票
22:10  阳台：翻看日程本，看了看远处城市的灯火
```

然后翻到明天：

```
明日
───
· 悬赏投票截止：后天 20:00
· 林教授说整理旧书的事，可以考虑一下
· 旅行商人预计明天下午到达
```

没什么命令。只是告诉你，明天有什么值得回来看看。

---

### 一周的节奏（中观循环）

**不是日历驱动，是事件驱动。** 一周没有固定的"周一发悬赏、周三投票、周五结算"。但有些自然规律：

**悬赏的典型周期**：
- 发布 → 提案期（2-3 天）→ 审议/讨论（1 天）→ 投票（2 天）→ 结果公示
- 一个悬赏从出题到结束，大约真实时间 3-5 天
- 不同悬赏可以重叠——同一时间可能有 2-3 个进行中的悬赏处于不同阶段
- 悬赏"节奏"由委员会掌控，但委员会不是机械地每周一扔题目，而是根据世界动态出题

**街区自身的节奏**：
- 每周有 1-2 次"高活跃窗口"——可能是某个提案的投票日，可能是旅行商人来的那天，可能是雨季开始的那天
- 这些高活跃窗口**不是每周固定日**。它们由正在发生的事决定
- 玩家的日程本会在高活跃窗口到来前给出预告

**一周中的"约定俗成"**：
- 虽然没有强制区分工作日和周末，但人的习惯会让某些时段更活跃——现实玩家大多在晚上和周末上线，这本身就会形成节奏
- 委员会可以考虑把重要投票截止日放在现实玩家的高在线时段
- 这不在设计文档里强制规定，留给阶段 2 社区实验观察

### 倒计时的节奏（宏观循环）

**"所有有时钟的地方，都有倒计时。"**

| 位置 | 呈现方式 |
|------|---------|
| 阳台屏幕 | 大号数字：**109,574 天** |
| 日程本角落 | 小号数字：**109,574d 07h** |
| 面包店墙上挂钟 | 表盘下面一行小字："剩余 109,574 天" |
| 街区议事厅 | 墙上的数字板，红色 |
| App 顶部 | 打开 App 时短暂显示，然后收起到角落 |
| 公园老树旁的日晷 | 日晷底座上刻着倒计天数（每天自动变化） |

**正常速度**：每天减少 1 天。玩家大多数时候不会注意，就像你不看手机右上角的电量——但你知道它在那里。

**跳秒机制**（倒计时加速或暂停）：

| 触发条件 | 效果 | 意义 |
|---------|------|------|
| 街区达到某个里程碑（第一次图书馆建成、第一次跨区合作） | 倒计时跳快 5 天 | 发展加速了时间——是好事 |
| 街区陷入严重分歧，投票参与率跌破 20% | 倒计时跳快 30 天 | 内耗在消耗人类的准备时间——不是惩罚，是反馈 |
| 文明级别的科技或文化突破 | 倒计时暂停 1 天 | 某个成就给了人类额外的喘息 |
| 外部信号异常（外星文明动向变化） | 倒计时跳快 X 天（数值不定） | 世界层事件——所有人同时看到 |

跳秒发生时，**日程本会震动。倒计时数字短暂变红。所有有时钟的地方同时变化。**

这不是灾难预警。这是——"你做的事，影响的不只是今天。"

---

### 时钟的两面

你在日程本里看到的倒计时，是正面——它在流逝。

但如果你在公园日晷上仔细看，底座的另一侧，刻着另一种文字：

> "你已在此 7 天。"

这不是倒计时。这是你来过多少天的记录。日历翻到这一页，也能看到。

**正面：还有多少天。**
**反面：你已经在这里多少天，留下了什么。**

因和果在同一只钟上。你做的每一件事，既改变你还能做什么（因），也定义了你已经做过什么（果）。

---

### 这一天结束后

玩家合上日程本。从阳台走进屋里。

远处的城市核心区仍然灯火通明。倒计时又跳了一格。不是催你，只是告诉你——时间在走。

明天的事，明天再说。

---

### 日常循环设计自查

对称核心原则：

- [x] 没有"每日任务""每周任务"——只有日程本，诚实记录，不命令
- [x] 不区分工作日和周末——但尊重人的自然节奏
- [x] 用事件推动时间——旅行商人来了就是一个事件，不是在"周三下午例行刷新"
- [x] 倒计时在所有钟表上，不张扬，但一直在
- [x] 工作不是生存——经营自己的店是自我表达，店开着或关着都是选择
- [x] 时钟的两面——正向流逝和反向积累，同时存在
- [x] 干净、纯粹、舒服——日程本不催你，世界不等你，但世界记得你
