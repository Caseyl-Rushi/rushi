# 如是 · 游戏设计文档
## 第三章：经济系统

> **核心原则**：这个世界没有氪金大佬。众生富足平等。你想做什么，需要消耗你的注意力和判断力——这些是稀缺资源。最终，你在这个世界最重要的印记，是你的贡献值，它转动文明的走向。

---

### 经济系统的三层结构

```
┌─────────────────────────────────────────┐
│  基础层：通用货币                          │
│  社交用。日常消费、交换、经营。              │
│  来源：经营产出 + 系统基础分配              │
├─────────────────────────────────────────┤
│  参与层：Token                            │
│  公共行为的门槛。投票、提案、发言、认可。      │
│  来源：系统每日/每周赠送 + 优质参与奖励       │
├─────────────────────────────────────────┤
│  最终层：贡献值                            │
│  转动文明走向的数值基础。                    │
│  来源：用 Token 参与后，产出被社区认可         │
└─────────────────────────────────────────┘
```

三层不可互相替代。通用货币买不了 Token。Token 买不了贡献值。贡献值不能当钱花。**三条线各自独立，但彼此咬合。**

---

### 第一层：通用货币

**定位**：这个世界的日常血液。你用它买面包、装修店铺、和邻居交换物资。它让社交有载体，但本身不带权力。

**获取途径**：

| 途径 | 说明 | 产出节奏 |
|------|------|---------|
| 经营产出 | 你的店铺/职业的日常运转产生货币 | 持续，根据经营状态波动 |
| 系统基础分配 | 每位居民每天自动获得一笔基本生活费 | 每日固定，人人等额 |
| 玩家间交换 | 你卖给邻居商品或服务，对方付你货币 | 即时 |
| 街区公共项目分红 | 如果街区有公共营利设施（如市集），居民获得分红 | 周期性 |

**消耗途径**：

| 途径 | 说明 |
|------|------|
| 购买商品 | 面包店的食品、杂货铺的用品、装饰材料 |
| 装修与升级 | 升级店铺、扩展经营空间 |
| 公共捐献 | 街区设施建设需要居民捐献建材或资金（自愿，非强制） |
| 部分 Token 相关功能 | 少数高消耗 Token 行为可能伴随通用货币消耗（如提交大型提案需要"场地费"） |

**通胀控制**：
- 系统基础分配是固定的，不随经济总量增长
- 公共捐献可以回流货币（货币从玩家回流到系统）
- 如果纸面验证发现通胀过快，委员会可以调整税率（见本章"税收"节）

**初始值建议**：
- 新居民入住房：赠送 500 通用货币
- 每日基础分配：50 通用货币
- 面包价格参考：5-15 货币 / 个
- 数值将在阶段 1 纸面验证中精确调整

### 第二层：Token

**定位**：公共行为的入场券。它代表的不是财富，是**你的注意力和判断力的稀缺性**。每天有限，每周补充，用不完不继承。你必须选择把有限的 Token 花在什么地方。

**设计哲学**：
- Token 不是付费货币。不能用真实货币购买。
- Token 是"参与配额"。防止垃圾信息、防止刷票、防止注意力通货膨胀。
- 类比 AI 的 token 收费模式——每次参与都有成本，这个成本逼迫你思考"什么是值得我投入的"。

**获取途径**：

| 途径 | 说明 | 额度 |
|------|------|------|
| 每日登录赠送 | 每人每天自动获得 | 100 Token（初设，待验证） |
| 每周补充 | 每周一自动获得 | 300 Token（初设，待验证） |
| 优质参与奖励 | 提案被采纳、发言被认可、悬赏完成 | 一次性奖励，额度不等 |
| 认可他人 | 给别人的贡献按"认可"，消耗 1 Token 但系统返还 2 Token | 反向激励——帮助社区发现有价值内容的人，不应该被惩罚 |
| 贡献值等级提升 | 贡献值每达到一个等级 → Token 日配额永久提升 | 如贡献值达 200 → 日配额从 100 升至 120 |

**消耗途径**（所有公共行为）：

| 行为 | Token 消耗（初设） | 说明 |
|------|-------------------|------|
| 投票（一次） | 10 | 每项提案可投一票 |
| 提交悬赏方案 | 50 | 提交完整方案，含文字说明 |
| 提交一般提案 | 30 | 非悬赏类的自发提案 |
| 议事厅发言（一次） | 5 | 在议事厅公开讨论中发表一次观点 |
| 认可他人（一次） | 1 → 系统返还 2 | 净赚 1 Token——鼓励价值发现 |
| 旁听议事厅 | 0 | 只听不说，不消耗 |
| 阅读公告板 | 0 | 信息获取不消耗 |
| 经营店铺 | 0 | 日常经营不消耗 Token |
| 聊天/社交 | 0 | 在街上和邻居自然对话不消耗 |

**消耗梯度**（同一天内重复操作的边际成本）：

| 第 N 次 | 投票 | 发言 | 提案 |
|---------|------|------|------|
| 1 | 10 | 5 | 30 |
| 2 | 10 | 5 | 50 |
| 3 | 15 | 10 | 80 |
| 4+ | 25 | 20 | —（当日不可再提） |

设计意图：你可以参与多项事务，但在同一件事上反复操作的边际成本递增。这防止了刷票，也让人在投票时更审慎——前三票便宜，后面的每一票都更贵。

**不累积原则**：
- 每日赠送的 Token 当天未用完 → 清零
- 每周补充的 Token 当周未用完 → 清零
- 不设 Token 存款。这是"使用权"，不是"资产"

例外：悬赏奖励的 Token 可以累积，不会过期。因为它们是你用成果换来的，不是系统施舍的。

**Token 配额提升机制**：

| 贡献值等级 | 日配额 | 周补充 | 备注 |
|-----------|--------|--------|------|
| 0-49 | 100 | 300 | 新居民 |
| 50-199 | 120 | 350 | 初露头角 |
| 200-499 | 150 | 400 | 活跃居民 |
| 500-999 | 180 | 500 | 核心贡献者 |
| 1000+ | 200 | 600 | 街区支柱 |

注意：配额提升幅度不大。一个贡献值 1000+ 的核心玩家，日配额只比新手多一倍。他不能垄断公共事务——他只是有更多空间参与。

### 第三层：贡献值

**定位**：你在这个世界最重要的印记。它不随时间贬值，不会消费掉。它衡量的是——**你对这个文明走向的累积影响**。

**获取途径**：

| 途径 | 贡献值奖励 | 条件 |
|------|-----------|------|
| 悬赏方案被投票选中 | 100-500（由悬赏难度决定） | 你的方案在投票中获得最高票 |
| 自提方案被社区投票通过 | 50-200 | 非悬赏类的自发提案被采纳 |
| 在重大事件中做出关键贡献 | 不定（系统中裁定） | 如紧急事件中你的建议被实施 |
| 持续高认可度产出 | 每月自动结算少量贡献值 | 你的提案或发言在过去一月内持续获得认可 |
| 担任委员会成员 | 每月固定贡献值 | 作为委员履职的回报（不多，象征性） |

**贡献值的不可消费性**：
- 贡献值不会因为你投票、发言、提交提案而减少
- 你的贡献值总量只增不减
- 它像你的"社会信用记录"——只看你做过什么，不看你"花了"什么

**贡献值的作用**：

| 作用 | 说明 |
|------|------|
| 投票权 | 贡献值决定你有几票（详见下文） |
| 提案门槛 | 提交公共提案需要最低贡献值（如 50） |
| Token 配额 | 贡献值等级决定每日 Token 配额 |
| 委员会资格 | 竞选委员会成员需要最低贡献值 |
| 街区迁移 | 搬到其他街区时，贡献值作为你在新街区的起始社会评价 |
| 文明遗产 | 一轮文明结束时，你的贡献值以某种形式继承到下一轮 |

**投票权对照表**：

| 贡献值 | 投票权 |
|--------|--------|
| 0-49 | 1 票 |
| 50-199 | 3 票 |
| 200-499 | 6 票 |
| 500-999 | 10 票 |
| 1000+ | 15 票 |

设计原则：边际投票权递减。一个 1000+ 贡献值的大佬有 15 票，但 500 个只有 1 票的普通居民加起来有 500 票。大佬可以说服，但不能碾压。

### 三层之间的流动关系

```
                系统基础分配
                     │
                     ▼
    ┌────── 通用货币 ──────┐
    │      （日常消费）      │
    │                      │
    │ 经营产出 ←→ 交换 ←→ 消费
    │                      │
    └──────────────────────┘
              │
              │ （货币不能换 Token，两条线独立）
              │
              ▼
    ┌────── Token ─────────┐
    │     （公共行为门槛）    │
    │                      │
    │ 每日赠送 → 花在投票/提案/发言
    │                      │
    │ 如果参与被认可 ──────→ 获得贡献值
    │                      │
    │ 贡献值提升 ──────────→ Token 配额提升
    └──────────┬───────────┘
               │
               ▼
    ┌────── 贡献值 ────────┐
    │   （文明走向的基础）    │
    │                      │
    │  不消费、不贬值          │
    │  只增不减              │
    │  你在这个世界最重要的印记  │
    └──────────────────────┘
```

关键约束：
- 通用货币 ↔ Token：不可互换
- Token ↔ 贡献值：Token 是入场券，贡献值是成绩单
- 贡献值 → Token：贡献值高了，每天能用的 Token 更多（配额提升）
- 贡献值 ✗ 通用货币：贡献值不能当钱花

### 税收系统

交换环节由系统自动征税。税率可调整。

**征税场景**：

| 场景 | 初设税率 | 说明 |
|------|---------|------|
| 玩家间交易 | 5% | 买家付 100，卖家收到 95，系统收回 5 |
| 公共市集交易 | 3% | 公共设施税率更低，鼓励在市集交易 |
| 跨街区交易 | 8% | 跨区交易税率更高，鼓励本街区经济内循环 |

**税收用途**：
- 回流系统，控制通胀
- 部分进入街区公共财政（用于公共设施维护、NPC 工资、公共项目补贴）
- 公共财政的使用需要街区投票决定

**税率调整权限**：
- 在小镇阶段，税率由 NPC 委员会设定（初设值）
- 进入大城市后，税率调整需要委员会提案 + 全民投票

### Token 费用参考：AI 收费模式对比

正如你提到的，这个世界的 Token 消耗设计参考了 AI token 的定价逻辑。

| 对比维度 | AI Token | 如是 Token |
|---------|---------|-----------|
| 本质 | 计算资源的消耗度量 | 注意力和判断力的消耗度量 |
| 获取 | 充值购买 | 系统赠送 + 优质参与奖励 |
| 消耗与产出的关系 | 消耗 token 换取 AI 输出 | 消耗 token 参与公共事务，产出可能被认可（获得贡献值）|
| 定价逻辑 | 输入便宜，输出贵 | 看便宜，说贵；投便宜，提贵 |
| 稀缺性 | 花完就没 | 花完就没（不累积） |

核心相似点：**token 像注意力预算。有限的资源逼迫你选择——什么是值得我花 token 的？**

### 数值验证要点（阶段 1 纸面验证时重点测试）

| 测试项 | 验证方法 |
|--------|---------|
| 新手 100 Token/天是否够用 | 模拟新手一天的标准参与（1 投票 + 1 发言 + 浏览公告板），看剩余比例是否 ≥ 30% |
| 消耗梯度是否有效防止刷票 | 模拟一个玩家试图在同一天投 10 次票，看边际成本是否阻止了他 |
| Token 配额提升是否显著但不泛滥 | 模拟贡献值 0→1000 的过程，看 Token 配额增长是否平滑 |
| 三层是否独立咬合 | 模拟一个玩家试图用通用货币买 Token、或用 Token 换贡献值，看系统是否有漏洞 |
| 税率对经济流通的影响 | 用 3% / 5% / 8% 三组分别跑 4 周，看哪个税率下交易最活跃且通胀可控 |
