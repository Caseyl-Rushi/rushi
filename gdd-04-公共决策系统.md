# 如是 · 游戏设计文档
## 第四章：公共决策系统

> **核心原则**：委员会发牌，玩家出牌，社区裁判。权力被拆成三份——出题权、答题权、裁判权——谁也不独占。一切公共行为消耗 Token，你的每一次参与都有重量。

---

### 决策系统的权力三角

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   委员会      │     │    玩家       │     │    社区       │
│  出题人       │     │   答题人       │     │   裁判        │
│              │     │              │     │              │
│ · 发布悬赏    │     │ · 提交方案    │     │ · 投票决定    │
│ · 设定奖励    │ ──→ │ · 发表观点    │ ──→ │ · 认可贡献    │
│ · 抛出议题    │     │ · 参与讨论    │     │ · 选出最优    │
│              │     │              │     │              │
│  不能裁判     │     │  不能出题     │     │  不能出题     │
└──────────────┘     └──────────────┘     └──────────────┘
         ↑                                       │
         └──────────── 投票结果反馈 ──────────────┘
```

### 一、委员会

#### 委员会的定位

委员会是**内容策展层**，不是权力中心。它的核心功能：出题、维持节奏、在僵局时仲裁。

委员会不做什么：
- 不评判方案好坏
- 不决定谁的方案通过
- 不干预玩家自发的提案
- 不控制街区的发展方向

委员会做什么：
- 发布悬赏任务
- 在信息噪音太大时过滤明显垃圾
- 在票数平局时打破僵局
- 维护规则的边界——确保提案不违反基本框架

#### 委员会的构成

**小镇阶段（NPC 代管）**：
- 委员会由 3-5 名 NPC 组成
- NPC 委员由系统设定，具有固定的"性格"和"倾向"
- 他们发布的悬赏题目由系统预设题库 + AI 辅助生成
- 他们对提案的仲裁（垃圾过滤/平局打破）遵循透明的规则

**城市阶段（人机混合）**：

| 席位类型 | 人数 | 产生方式 | 任期 |
|---------|------|---------|------|
| 玩家委员 | 3-5 人 | 全民选举 | 一季（真实时间约 3 个月） |
| 系统委员 | 2 人 | 系统固定（开发者/运营） | 永久 |
| 专业委员 | 0-2 人 | 邀请制（真实世界的专家/创作者） | 不定 |

开发者保留固定席位的目的是：确保底线不被突破。即使所有玩家委员都失职，系统委员仍然能维持基本运转。

#### 委员会的权限和制约

| 权限 | 条件 | 制衡 |
|------|------|------|
| 发布悬赏 | 自由 | 悬赏如果连续两次参与率低于 30%，该委员的选题权暂停一周 |
| 过滤垃圾内容 | 需 2/3 委员投票 | 被过滤者可以申诉，由全民投票复核 |
| 打破平局 | 投票结果平局时 | 委员会打破平局的决定本身可以被联署挑战 |
| 调整 Token 基础配额 | 需全体委员 3/4 通过 | 调整幅度有上限（如不超过 20%），且每年只能调整一次 |

#### 委员的罢免与轮换

- 玩家委员任期结束自动轮换
- 任职期间如果有 30% 居民联署不信任，触发罢免投票
- 罢免投票需要 60% 支持率通过
- 被罢免的委员在接下来一个任期内不得再次参选

### 二、悬赏系统

悬赏是公共决策的**主要入口**。大部分公共议题由委员会以悬赏形式抛出。

#### 悬赏的类别

| 类别 | 示例 | 典型周期 | 贡献值奖励 |
|------|------|---------|-----------|
| 街区建设 | 西侧空地开发方案、街区公园改造 | 5 天 | 200-500 |
| 文化/社会 | 街区标志设计、节日策划、社区公约修订 | 3-4 天 | 100-300 |
| 紧急响应 | 雨季设施加固、突发事件应对 | 1-3 天 | 200-400 |
| 跨区事务 | 与相邻街区的合作方案、资源共享提案 | 5-7 天 | 300-600 |
| 城市级 | 城市交通规划、重大科技投资方向 | 7-10 天 | 500-1000 |
| 开放式 | "我们街区目前最需要什么？"——无预设方向 | 4 天 | 150-300 |

#### 悬赏的完整流程

```
发布 ──→ 提案期 ──→ 审议期 ──→ 投票期 ──→ 公示 ──→ 执行 ──→ 验收
  │        │          │          │         │        │        │
  │    2-3天     1天        2天      1天    视项目   建成后
  │        │          │          │         │        │
  │   玩家提交     议事厅     全民投票   结果公示   资源到位   项目上线
  │   方案文字     公开讨论               不可更改   开工建设   数据反馈
```

**提案期**（2-3 天）：
- 玩家通过 App 提交方案
- 方案格式：文字描述 + 可选附图（简单示意图）
- 提交消耗 50 Token
- 提案期结束后，所有有效方案进入审议

**审议期**（1 天）：
- 议事厅公开讨论（App 内有专用讨论区）
- 提案人可以修改方案细节、回应质疑
- 讨论区发言消耗 5 Token / 次
- 审议结束后，方案锁定，进入投票

**投票期**（2 天）：
- 全民投票。每票消耗 10 Token
- 投票权取决于贡献值（见第三章贡献值-投票权对照表）
- 投票为单选
- 投票结束后，得票最高者通过

**公示期**（1 天）：
- 结果在公告板和 App 公示
- 如有异议（如发现方案抄袭、违规），可在公示期申诉
- 申诉由委员会仲裁

**执行期**（视项目而定）：
- 被采纳的方案进入施工/实施
- 玩家可以在街上看到变化——空地围挡、建筑渐起
- 可能触发执行中的新事件——预算超了、材料缺了、有人自愿帮忙

**验收**：
- 项目建成，投入运营
- 运营数据反馈在街区繁荣指数中
- 提案人获得贡献值奖励

### 三、玩家自提方案

除了回应委员会的悬赏，玩家也可以**自己发起提案**。

#### 自提方案的条件

| 条件 | 说明 |
|------|------|
| 贡献值 ≥ 50 | 防止完全的新手随意提案 |
| 消耗 30 Token | 提交方案的基础成本 |
| 需附简要理由 | 为什么这个提案值得街区关注 |
| 每周限提 2 次 | 防止提案泛滥 |

#### 自提方案的处理流程

1. 玩家提交自提方案
2. 方案进入公告板的"自发提案"区域
3. 公示 2 天。在此期间：
   - 其他居民可以留言支持或质疑
   - 如果有 ≥ 10 人或 ≥ 10% 的居民（取较大值）联署支持，提案进入正式投票
4. 如果公示期内联署不足，提案自动归档，不进入投票
5. 进入投票后，流程与悬赏方案相同

#### 自提方案的意义

自提方案是委员会的"反向力量"。委员会出的题目可能偏颇——他们问的东西不一定就是居民最关心的。自提方案让任何有足够社区支持的人，都能把议题摆上台面。

### 四、投票机制

#### 基本投票规则

- 每项提案单独投票，单选
- 每票消耗 10 Token
- 投票权由贡献值决定（见第三章）
- 投票为记名投票——你的投票记录公开可查。这是公共决策，不是匿名投票

#### 为什么是记名投票

匿名投票看似公平，但在这个世界里，**你的选择应该被看见**。你投了图书馆反对市集——这是你的判断。你不需要向任何人解释，但你需要为你的选择负责。如果事后证明你的选择伤害了街区，下次投票时其他居民可能会更审慎地看待你的判断。

记名投票和贡献值一起，构成了你在公共决策中的**完整人格**——不是匿名的数字，而是有立场、有判断力、有历史的参与者。

#### 特殊投票规则

**紧急投票**（适用于紧急响应类悬赏）：
- 投票期缩短至 4-12 小时
- 投票消耗 Token 翻倍（20 Token / 票）
- 需要 ≥ 30% 居民参与才有效

**跨街区投票**：
- 涉及两个或以上街区的事务
- 每个街区的投票结果按街区人口加权
- 你在外街区投票，你的贡献值只算 70% 权重

### 五、议事厅

议事厅是公共讨论的物理/数字空间。它有两层形态：

**第一层：App 内的讨论区（日常）**

- 每个悬赏和提案都有专属讨论区
- 玩家可以发表观点、回复他人、提出问题
- 每次发言消耗 5 Token——不是为了限制表达，是为了让每一句话都有分量
- 高质量的发言可以被"认可"，发言者获得返还 Token（系统奖励）
- 讨论区在审议期最活跃，但不关闭——方案建成后人们仍然可以反馈

**第二层：物理议事厅（关键时刻）**

- 街区内有一栋建筑，即议事厅
- 平时可以进去坐坐，看历史提案记录
- 重大投票日，议事厅内可以看到实时投票数据和讨论热度
- 当讨论区出现高质量辩论，议事厅建筑上方会在街景中显示一面小旗（视觉提示）——"里面正在聊好东西"
- 进入议事厅不消耗 Token。听别人说话是免费的

### 六、决策对世界的影响

每一个通过的决策，不只改变内容，也改变数据。

#### 直接影响

| 决策类型 | 影响的世界指标 |
|---------|-------------|
| 建设提案（图书馆、公园、市集） | 街区繁荣指数的对应分项（文化/宜居/经济）|
| 规则提案（社区公约、经营规范） | 影响居民行为的约束条件 |
| 合作提案（跨区协议、资源共享） | 街区之间的关系值、资源流动 |
| 紧急响应 | 灾害损害程度、恢复速度 |

#### 连锁反应

一个决策不是终点。它是新决策的起点：

- 图书馆建成 → 文化指数上升 → 更多文化型居民被吸引入住 → 新的文化类提案出现 → 图书馆需要扩建
- 市集建成 → 经济流通增加 → 税收增加 → 公共财政充裕 → 可以建更多公共设施
- 公园建成 → 宜居指数上升 → 居民满意度提升 → 人口增长 → 现有设施压力增大 → 需要新的提案

**任何一个决策都在为下一个决策铺路，或挖坑。** 玩家在投票时面对的，不只是"建什么好看"，而是"建什么以后，这个街区往哪里走"。

### 七、垃圾信息和恶意行为的防护

| 威胁 | 防护机制 |
|------|---------|
| 提交垃圾方案（灌水、测试） | Token 消耗（50 Token/次）+ 委员会可标记为垃圾，标记三次后提案功能暂停 3 天 |
| 刷票 | 消耗梯度（第四票起 25 Token/票）+ 投票记录公开可查 |
| 恶意认可（互相刷认可换 Token） | 认可他人的 Token 返还很少（1→2）。且认可记录公开——如果两个人互相疯狂认可，会被看到 |
| 提案抄袭 | 公示期可以揭发。核实后在提案人记录中标记，影响贡献值获取 |
| 小团体垄断投票 | 贡献值的边际投票权递减（500 个 1 票的人 > 1 个 15 票的人）|

- [x] 记名投票——你的判断有迹可循
- [x] Token 消耗让每次参与都有成本
- [x] 消耗梯度防止刷票
